System Testing Instructions.

A step-by-step guide to validating the UniCore ERP architecture, from database provisioning to cross-institutional lead routing.

01

Environment Setup

To begin the demo, you must first provision the database and server folders.

Step 1.1: Run setup_demo.php

Navigate to http://localhost/unicore_erp/setup_demo.php. This script builds the unicore_db, seeds data, and creates the /uploads/docs/ directory.

Step 1.2: Admin Authentication

Login at login.php using admin / admin123. This account is pre-mapped to the Superadmin role.

02

RBAC Governance

UniCore V2.0 uses a Dynamic Access Matrix.

  • Navigate to Access Matrix in the sidebar.
  • Toggle a permission (e.g., social_router) for a role.
  • The system uses AJAX to sync the role_permissions table instantly.
03

Lead Ingestion & Grouping

Test the "Single Group" logic for multi-course applicants.

Scenario: The John Doe Duplicate

"John applies for Engineering via WhatsApp, then applies for Business via API using the same phone number."

The system detects the duplicate phone, assigns a group_tag, and creates two distinct leads. This prevents data overwriting while keeping the student's profile linked.

04

Compliance Terminal

Verify documents and monitor engagement.

Document Verification

Attach a sample file to 'High School Marksheet'. Once 4 docs are 'Verified', the system unlocks the 'Finalize' button.

Group Awareness

Notice the "Multi-Lead Profile" badge. Clicking the "Also Applied" chips switches the terminal between John's two applications.
